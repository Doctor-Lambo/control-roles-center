# Contributing

Спасибо за интерес к проекту! Вклад в любом виде приветствуется.

## Баг-репорты

Перед созданием Issue проверьте, что такой проблемы ещё нет в [открытых Issues](../../issues).

В баг-репорте укажите:
- версию расширения (см. синоним или `CHANGELOG.md`);
- версию платформы 1С и режим запуска (файловый / клиент-серверный);
- версию конфигурации, к которой подключено расширение;
- шаги воспроизведения;
- ожидаемое и фактическое поведение;
- скриншот / лог (если применимо).

## Предложения новых возможностей

Откройте Issue с меткой `enhancement`. Опишите:
- сценарий использования (какую задачу решает);
- предлагаемое поведение;
- альтернативы, которые рассматривались.

## Pull requests

1. Перед большим PR лучше открыть Issue для обсуждения подхода.
2. Форкните репозиторий и создайте ветку от `main`:
   ```
   git checkout -b feature/<short-name>
   ```
3. Соблюдайте стиль кода (см. раздел ниже).
4. Обновите `CHANGELOG.md` в секции `[Unreleased]`.
5. Откройте PR с понятным описанием: что и зачем.

## Стиль коммитов

Формат: `<тип>: <краткое описание>`

Типы:
- `feat` — новая функциональность;
- `fix` — исправление бага;
- `docs` — только документация;
- `refactor` — рефакторинг без изменения поведения;
- `perf` — оптимизация производительности;
- `chore` — сервисные изменения (зависимости, сборка, CI).

Примеры:
- `feat: добавлено заполнение прав по пользователю`
- `fix: исправлена сборка CFE при отсутствии шаблонов ограничений`
- `docs: обновлён быстрый старт в README`

## Стиль кода

- Язык: **BSL**, комментарии и имена — по-русски (в соответствии со стандартами 1С).
- Префикс всех новых объектов метаданных: `цур_`.
- Отступы — табы.
- Длина строки — до 120 символов.
- Для публичных процедур/функций обязателен заголовок с описанием параметров и возвращаемого значения.
- Запросы — в формате с переносом строк и `|` (см. существующий код).
- Не использовать `.Реквизит` на ссылках — применять `ОбщегоНазначения.ЗначениеРеквизитаОбъекта` и аналоги.

Полный набор правил — в стандартах разработки 1С (ИТС).
