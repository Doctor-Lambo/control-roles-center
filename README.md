## О проекте

**Центр управления ролями** (ЦУР) — расширение 1С:Предприятие 8.3 для конструирования пользовательских ролей в режиме предприятия без изменения основной конфигурации. Позволяет:

- проектировать роли в пользовательском режиме с детальной настройкой прав доступа и RLS;
- заполнять права из существующих ролей конфигурации, профилей групп доступа и пользователей;
- автоматически генерировать расширение (`*.cfe`), содержащее спроектированные роли и adopted-объекты;
- управлять правами сразу для нескольких информационных баз из одной точки.

Проект вдохновлён и частично основан на [Custom Role Generator](https://github.com/rarus/custom-role-generator) компании [1С-Рарус](https://rarus.ru/), распространяется под совместимой лицензией CC BY-SA 4.0.

## Возможности

- **Конструктор ролей** — графический редактор прав доступа на все виды метаданных
- **RLS с шаблонами** — поддержка ограничений доступа к данным на уровне полей и строк, с возможностью использовать шаблоны ограничений
- **Заполнение прав** — из ролей конфигурации, из профилей групп доступа (БСП), из конкретных пользователей
- **Загрузка XML ролей** — прямой импорт XML-описаний ролей типовой конфигурации в базу
- **Мульти-ИБ** — хранение и ведение прав для нескольких целевых информационных баз
- **Генерация CFE** — автоматическая сборка расширения с adopted-объектами, автоинкремент версии
- **Режимы редактирования** — простой (для обычных пользователей) и экспертный (для разработчиков)
- **Работа без БСП** — не требует БСП, но при её наличии можно подключить отдельные методы через настройки

## Требования

| Требование | Значение |
|------------|----------|
| Платформа | 1С:Предприятие 8.3.24 и выше |
| Режим совместимости расширения | 8.3.24 |
| Конфигурация | любая на управляемых формах |
| БСП | опционально (для заполнения прав из профилей групп доступа) |

## Быстрый старт

1. Подключить CFE к целевой базе и обновить конфигурацию (F7).
2. Настроить определяемые типы, `Настройки ЦУР` и информационную базу сборки — [docs/USAGE.md#подготовка](docs/USAGE.md#подготовка).
3. Создать роль и сгенерировать CFE — [docs/USAGE.md#создание-роли](docs/USAGE.md#создание-роли).

## Использование

Базовый сценарий: [создать роль](docs/USAGE.md#создание-роли) → [заполнить права](docs/USAGE.md#заполнение-прав-из-источников) из источника или вручную → [настроить RLS](docs/USAGE.md#работа-с-rls) при необходимости → [сгенерировать CFE](docs/USAGE.md#генерация-расширения-cfe) → установить расширение в целевую базу.

Полное руководство со всеми сценариями, настройками, RLS, шаблонами ограничений и FAQ — в [**docs/USAGE.md**](docs/USAGE.md).

## Отличия от оригинального проекта

Форк и развитие [Custom Role Generator](https://github.com/rarus/custom-role-generator) от компании [1С-Рарус](https://rarus.ru/):

- загрузка XML ролей конфигурации в базу (не только во внешний файл);
- заполнение прав по пользователю и профилю групп доступа;
- нет ограничения в 99 999 строк на роль — права хранятся в регистре сведений, а не в табличной части;
- расширенная поддержка RLS — поля и шаблоны ограничений;
- автоинкремент версии расширения;
- поддержка нескольких целевых ИБ одновременно.

## Планы развития

- Выделение в самостоятельную конфигурацию-библиотеку
- HTTP-сервис для удалённых баз (получение ролей, профилей, деплой CFE)
- Ручное управление правами на объектах и группах объектов
- Прогресс фоновых заданий
- Хранение метаданных целевой конфигурации для работы из отдельной базы
- Загрузка ролей из Git-репозитория
- Перевод интерфейса на английский
- Полноценная поддержка Linux
- Добавление ролей в существующие подсистемы конфигурации
