## Подготовка

### Шаг 1. Заполнение определяемых типов

В конфигураторе целевой базы:

1. Открыть **Определяемые типы**
2. Указать:
   - `цур_Пользователи` → `СправочникСсылка.Пользователи`
   - `цур_ПрофилиГруппДоступа` → `СправочникСсылка.ПрофилиГруппДоступа`
3. Сохранить конфигурацию и обновить (F7)

Если в базе нет справочника `ПрофилиГруппДоступа` (например, БСП не подключена) — оставьте второй определяемый тип пустым, заполнение прав из профилей будет недоступно.

### Шаг 2. Первый запуск

При первом запуске расширение автоматически:
- заполняет регистр сведений `Реестр видов метаданных` (список видов объектов платформы с их доступными правами);
- выставляет флаг `Инициализация выполнена`.

Повторный запуск инициализации можно вызвать из формы настроек.

## Настройки

Открыть: **панель разделов → `Настройки ЦУР`** (или общая команда `цур_Настройки`).

### Параметры

| Параметр | Описание |
|----------|----------|
| **Путь к платформе 1С** | Полный путь к `1cv8.exe` (для автоматической выгрузки конфигурации целевой ИБ). Пример: `C:\Program Files\1cv8\8.3.24.xxxx\bin\1cv8.exe` |
| **Режим загрузки ролей** | Способ получения XML ролей: из файла, автоматическая выгрузка, из регистра |
| **Каталог выгрузки конфигурации** | Путь, куда `DESIGNER` выгружает XML при автоматическом режиме |
| **Использовать БСП** | Включает интеграцию с подсистемой «Пользователи» БСП (заполнение прав по пользователю/профилю) |
| **Использовать промежуточную базу** | Использовать отдельную файловую ИБ как промежуточный кэш для операций сборки |

### Режимы загрузки ролей

1. **Из каталога файлов** — XML-файлы ролей берутся из заранее выгруженного каталога `DumpConfigToFiles`
2. **Выгрузить автоматически** — расширение само запускает `DESIGNER` и выгружает нужные роли
3. **Из регистра** — XML ролей уже загружены в регистр `XML ролей конфигурации`

## Информационные базы сборки

Справочник `Информационные базы` хранит целевые ИБ, для которых собираются расширения с ролями.

### Создание элемента

1. Открыть справочник `Информационные базы`
2. `Создать` → заполнить:
   - **Наименование** — произвольное (например, «ERP Продакшн»)
   - **Тип** — файловая / серверная
   - **Строка подключения** — путь к `1Cv8.1CD` или `Srvr="…";Ref="…"`
   - **Использовать для сборки** — флаг активной ИБ (только одна ИБ может быть активной)
3. Записать

### Активная ИБ

Все операции (загрузка XML, генерация CFE, заполнение прав из источников) выполняются для ИБ со флагом `Использовать для сборки`. Для переключения — снять флаг с одной ИБ и поставить на другой.

> ⚠️ **Безопасность учётных данных.** Пароль в реквизите `Пароль` сохраняется в СУБД **без шифрования** (свойство `passwordMode` маскирует только ввод в форме). Используйте отдельную техническую учётную запись с минимальными правами — только чтение метаданных, без доступа к чувствительным данным. Административные учётные записи применять **не рекомендуется**. Подробнее — в разделе [Предупреждение и ответственность](../README.md#предупреждение-и-ответственность).

## Загрузка XML ролей конфигурации

Перед созданием новых ролей нужно загрузить XML-описания существующих ролей конфигурации — чтобы можно было ссылаться на них при заполнении прав и использовать шаблоны ограничений.

### Обработка «Загрузка XML ролей»

Открыть: **обработка `цур_ЗагрузкаXMLРолей`**.

Варианты загрузки:

1. **Из файлов** — указать каталог с выгруженной конфигурацией (`DumpConfigToFiles`), выбрать нужные роли, нажать `Загрузить`.
2. **Автоматически** — расширение само запустит `DESIGNER` для активной ИБ и выгрузит все роли. Требуется заполненный `Путь к платформе` в настройках.

После загрузки XML попадают в регистр `XML ролей конфигурации` и становятся доступны:
- для выбора при заполнении прав новой роли;
- для извлечения шаблонов ограничений;
- для включения в собираемое расширение.

## Создание роли

### Простой режим

1. Открыть справочник `Роли` → `Создать`
2. Заполнить:
   - **Имя** — латиницей, как в конфигураторе (например, `ПолныеПрава_Офис`)
   - **Синоним** — человекочитаемое название
3. В табличной части **Права**:
   - выбрать объект метаданных через форму выбора;
   - выставить флажки прав (Чтение, Добавление, Изменение, Удаление, Просмотр, Редактирование, …);
   - (опц.) открыть редактор ограничений и прописать RLS.
4. `Записать и закрыть`

### Экспертный режим

Для массовой правки:
1. В форме роли переключиться в режим `Эксперт`
2. Открыть регистр `Права доступа` с отбором по роли
3. Править строки прав напрямую

### Статус роли

Поле `Статус` (перечисление `цур_СтатусыРоли`):
- **Черновик** — роль в разработке, не попадает в сборку по умолчанию
- **Готова** — роль включается в сборку CFE
- **Архив** — роль исключена, но не удалена

## Заполнение прав из источников

Кнопка `Заполнить` в форме роли → выбор типа источника (перечисление `цур_ТипыИсточниковЗаполнения`):

| Источник | Что копируется | Требования |
|----------|---------------|------------|
| **Роль конфигурации** | Все права из загруженной XML-роли | XML роли должна быть загружена |
| **Профиль групп доступа** | Объединение прав всех ролей профиля | БСП + заполненный определяемый тип `цур_ПрофилиГруппДоступа` |
| **Пользователь** | Объединение прав всех профилей и ролей пользователя | БСП + заполненный `цур_Пользователи` |

Режимы слияния:
- **Заменить** — очистить текущие права и подставить источник
- **Добавить** — объединить с существующими (логическое ИЛИ)
- **Вычесть** — убрать права, которые есть в источнике

## Работа с RLS

RLS (Record Level Security) — ограничения доступа к строкам таблиц на уровне SQL.

### Добавление ограничения

1. В форме роли в строке права нажать на колонку **Ограничение доступа**
2. Откроется редактор `цур_РедакторТекстаОграниченияДоступа`
3. Прописать текст ограничения — стандартный синтаксис 1С для RLS:
   ```
   ГДЕ Контрагент.Основной = ИСТИНА
      И Организация В (&Организация)
   ```
4. Проверить синтаксис (кнопка `Проверить`)
5. `ОК`

### Доступные конструкции

- Поля объекта через `.` (например, `Контрагент.ИНН`)
- Параметры сеанса: `&ТекущийПользователь`, `&Организация`
- Функции глобального контекста
- Шаблоны ограничений: `#ПоЗначениям(...)`, `#ПоТиповымЗначениям(...)` — см. раздел ниже

### Редактирование полей RLS

Если ограничение ссылается на поля — в отдельной вкладке формы редактора можно указать список полей, к которым применяется ограничение (вместо «Все поля»).

## Шаблоны ограничений доступа

Регистр `Шаблоны ограничений` хранит переиспользуемые шаблоны RLS.

### Создание шаблона

1. Открыть регистр `Шаблоны ограничений`
2. Добавить запись:
   - **Роль** — к какой роли привязан шаблон
   - **Имя шаблона** — идентификатор (например, `ПоОрганизации`)
   - **Текст шаблона** — параметризованный SQL-текст
   - **Параметры** — список параметров шаблона

### Использование в ограничении

В тексте RLS:
```
#ПоОрганизации("Организация")
```

При сборке CFE шаблон разворачивается в полный текст ограничения.

## Генерация расширения (CFE)

Обработка **`Генератор ролей`**.

### Шаги

1. Открыть `цур_ГенераторРолей`
2. Проверить активную ИБ (поле в шапке формы)
3. В табличной части выбрать роли для включения:
   - по умолчанию — все роли со статусом «Готова»;
   - можно снять/поставить галочки вручную.
4. Указать путь выгрузки CFE (например, `C:\Releases\ЦУР_Роли_v1.2.cfe`)
5. Нажать `Сформировать`

### Что делает генератор

1. Создаёт расширение-заготовку из шаблона `гпр_ШаблонРасширения`
2. Для каждой выбранной роли формирует XML-описание через `цур_ПостроительXMLРолейСервер`
3. Добавляет adopted-объекты (метаданные, на которые ссылаются права)
4. Инкрементирует версию расширения
5. Собирает CFE через `DESIGNER` и сохраняет в указанный путь

### Установка полученного CFE

В конфигураторе целевой ИБ: **Конфигурация → Расширения конфигурации → Добавить из файла** → выбрать собранный CFE → обновить (F7).

## Работа с несколькими информационными базами

Для каждой ИБ ведётся свой набор:
- загруженных XML ролей;
- пользовательских ролей;
- прав доступа;
- шаблонов ограничений.

### Переключение между ИБ

1. Открыть `Информационные базы`
2. Снять флаг `Использовать для сборки` с текущей активной ИБ
3. Поставить флаг на нужной
4. Все операции (загрузка, генерация) теперь выполняются для новой активной ИБ

### Копирование ролей между ИБ

Прямого инструмента переноса роли из одной ИБ в другую в 2.2.0.1 нет. Обходное решение: экспорт/импорт через XML.

## FAQ

**Работает ли без БСП?**
Да. Только функции «Заполнить по пользователю / профилю» требуют БСП. Остальное работает автономно.

**Что делать, если в рейтинг выгрузки попадают не все объекты метаданных?**
Проверить регистр `Реестр видов метаданных` — там настраивается список видов, поддерживаемых ЦУР. Если какого-то вида нет — добавить вручную.

**Почему сгенерированный CFE не устанавливается в целевую базу?**
- Проверить, что режим совместимости CFE совпадает с платформой целевой базы (по умолчанию 8.3.24).
- Убедиться, что в CFE нет ролей с именами, конфликтующими с типовыми.
- Проверить версию расширения — если не инкрементировалась, установка нового CFE будет отклонена.

**Как восстановить права после сбоя генерации?**
Все права хранятся в регистре `Права доступа` и сохраняются между попытками сборки. Генерация не модифицирует пользовательские данные, только читает.

**Где посмотреть текущую версию расширения?**
В свойствах конфигурации расширения — поле `Версия`. Автоинкремент выполняется при каждой успешной сборке.

**Как очистить загруженные XML ролей?**
Обработка `Загрузка XML ролей` → кнопка `Очистить регистр` (экспертный режим).

**Совместимо ли с Linux?**
Частично. Операции работы с файлами используют системные разделители; запуск `DESIGNER` через `1cv8` поддерживается. Для стабильной работы под Linux требуется дополнительное тестирование (в планах развития).
